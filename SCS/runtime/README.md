# Runtime Model Integration

这个目录用于给 SCS 增加多模型接入层。

## 当前目标

让 SCS 从“文档 + 模板仓库”继续向“可接多模型的运行仓库”推进。

## 目录说明

- `model_registry.json`：所有可用模型的总表
- `role_model_map.json`：SCS 角色到模型的映射
- `routing_policy.json`：不同阶段/任务的切换规则
- `providers/`：不同 provider 的连接配置模板

## 推荐思路

- 总控 / 终局 / 稽核 / 深层治理：优先用长上下文、逻辑稳定模型
- 正文写作 / 文风润色：优先用文笔更强的模型
- 摘要 / 格式化 / 低价值重复步骤：可用低成本模型

## 当前默认建议

最高级新书栈建议使用：

- `V1.0`
- `V1.1a`
- `V1.1b`
- `V1.1c`
- `V1.2b`

旧项目迁移层 `V1.2a` 不建议默认带入新书栈。
