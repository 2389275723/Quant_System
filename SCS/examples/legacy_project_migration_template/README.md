# Legacy Project Migration Template

这是一个用于把旧项目、半途项目、老坑接回 SCS 的示例目录。

## 推荐栈
- V1.0 Core
- V1.2a Legacy & Migration Layer
- V1.1b Workbench Layer

## 使用顺序
1. 填写 `legacy_project_intake.example.json`
2. 填写 `legacy_route_reconstructor.example.json`
3. 填写 `foundation_backfill_builder.example.json`
4. 填写 `chapter_history_reconstructor.example.json`
5. 填写 `legacy_issue_scan.example.json`
6. 填写 `legacy_payoff_recovery_map.example.json`
7. 填写 `legacy_finale_recovery_map.example.json`
8. 填写 `migration_mode_selector.example.json`
9. 生成 `legacy_migration_summary.example.json`
10. 通过 `legacy_reentry_gate.example.json` 后再接回主流程

## 适合
- 已经写了很多章但没有系统化管理的项目
- 想救旧坑的项目
- 想判断终局还能不能收的项目
