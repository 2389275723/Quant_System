# FAQ

## 1. 现在这个仓库能直接拿来开项目吗？
可以。
当前仓库已经具备：
- 方法论文档
- 模板
- 预设实战包
- 多题材 bootstrap packs

最适合的默认开法是：
**V1.0 + V1.1a + V1.1b**

## 2. 我应该先看哪里？
优先顺序：
1. `START_HERE.md`
2. `docs/getting-started.md`
3. `docs/recommended-stacks.md`
4. 选一个 `presets/`

## 3. 新项目默认用哪一套？
默认先用：
- `presets/new_project_standard.json`

## 4. 复杂长篇什么时候加深层治理？
当项目出现以下任一情况时，建议加 `V1.1c`：
- 章数上来后人物开始漂
- 世界开始变重
- payoff 开始同型化
- mixed 项目副轴越来越强

## 5. 旧项目怎么接进来？
直接走：
- `presets/legacy_project_migration.json`
- `docs/legacy-migration.md`

## 6. 需要一上来全开完整治理包吗？
不建议。
大多数项目先用标准包更合理，复杂度上来再逐步加层。

## 7. 这个仓库现在的定位是什么？
它现在是一个：
**以文档、模板、预设实战包为核心的首版 SCS 仓库包**。
